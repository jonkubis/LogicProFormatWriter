# Donor sessions & the embedded seeds — provenance + how to regenerate

The library generates Logic `.logicx` sessions with **no loose donor bundles at runtime**:
it ships a small `logicx/data/` directory (~60 KB gzipped) derived from a handful of donor
`.logicx` sessions. This file is the map back — **what each donor is, how to remake it in
Logic, what gets extracted from it, and how to rebuild `logicx/data/`.**

## The golden rule

```
fixtures/ + templates/  =  SOURCE OF TRUTH (the donor .logicx sessions; keep them)
        │   python3.12 bake_seeds.py          (the ONLY way data/ is produced)
        ▼
logicx/data/  =  GENERATED ARTIFACTS  (gzipped seeds; never hand-edit — regenerate instead)
```

To update anything baked: re-make / replace the donor `.logicx` (steps below), then run
`python3.12 bake_seeds.py`. It re-derives every `logicx/data/` file and self-verifies the
round-trip. `bake_seeds.py` `DONORS = {...}` holds the exact fixture paths.

## The donors

All made in **Logic Pro 12.0.1**. A "settled" save = open the project and ⌘S once (Logic
trims the pre-allocated mixer on re-save — synthesize on the FRESH donor, not a re-save).

| key (in `bake_seeds.py`) | donor `.logicx` | how to remake it in Logic | feeds |
|---|---|---|---|
| `audio_base` | `fixtures/lots of audio tracks/1 from 64 audio tracks.logicx` | New empty project → add **64 audio tracks** → **delete 63** (keep 1). This keeps the 64-channel *pre-allocated mixer* (63 free slots to activate). Save. | `logicx/data/audio_base.seed` |
| `mixed_base` | `fixtures/midi test/mixed_template.logicx` | New project → add **1 Software Instrument track + 1 Audio track**. Save. | `logicx/data/mixed_base.seed` |
| `mixed_1inst` | `fixtures/midi test/mixed_template + 1 inst.logicx` | Open `mixed_template` → add **one** Software Instrument track → Save As. | `infra.json.gz` → instrument heavy-op |
| `mixed_2inst` | `fixtures/midi test/mixed_template + 2 inst.logicx` | Open `+ 1 inst` → add **one more** instrument track → Save As. | `infra.json.gz` → instrument UCuA v2 |
| `mixed_1audio` | `fixtures/midi test/mixed_template + 1 audio.logicx` | Open `mixed_template` → add **one** Audio track → Save As. | `infra.json.gz` → audio heavy-op |
| `f23_midi_proto` | `templates/F23_av.logicx` | A combined session: a few audio tracks (each with an audio region) **+** a few instrument tracks (each with an **empty** MIDI region named `Inst 1`…) **+** a marker track. | `infra.json.gz` → MIDI-region prototype |
| `f21_audio_proto` | `templates/F21_multitrack_regions.logicx` | A multi-track session with **≥1 audio region** placed (the region clone source). | `infra.json.gz` → audio-region prototype |

## What `logicx/data/` contains (all regenerated by `bake_seeds.py`)

- **`audio_base.seed`** / **`mixed_base.seed`** — a packed (gzipped) set of the donor
  bundle's files: `ProjectData`, `MetaData.plist`, `ProjectInformation.plist`,
  `DisplayState.plist`, `DisplayStateArchive`. The cosmetic **`WindowImage.jpg` (154 KB) is
  dropped**. These are the bases the synthesizers mutate and the bundle assembler writes.
- **`infra.json.gz`** — **pre-extracted constant records**, JSON+base64 (human-inspectable),
  with a `_provenance` block naming the donor + extractor for each piece:
  - `instrument_infra` ← `instrument_infrastructure(mixed_1inst, mixed_2inst, mixed_base)`
    (the new-instrument ivnE, UCuA v1/v2, the 0x4000000 trio, the grown 241-byte strip).
  - `audio_infra` ← `audio_infrastructure(mixed_1audio, mixed_base)`.
  - `midi_region_proto` ← `_midi_region_prototype(f23_midi_proto)` (the 5-record region
    group `tSxT/lytS/qeSM/karT/qSvE` + the `0x20` placement event).
  - `audio_region_proto` ← `extract_audio_region_prototype(f21_audio_proto)` (the region-0
    `gRuA/lFuA` group + the `0x24` placement event).
  This is what lets us **not ship the 5 reference donors at runtime** — only the few KB of
  records the extractors pull (the full sessions stay in `fixtures/`/`templates/` for
  regeneration + the test suite).

## If a baked piece ever looks wrong

1. The donor is the source — open the relevant `.logicx` in `fixtures/`/`templates/`,
   confirm it matches the "how to remake" recipe (and the Logic version).
2. `python3.12 bake_seeds.py` to re-derive `logicx/data/` (it self-verifies round-trips).
3. The byte-level meaning of every extracted record is in `PROJECTDATA_FORMAT.md`
   (§10.6–§10.9 for tracks/regions/synthesis, §8/§8.1 for the audio bin).
